DocSeek — Acceptance test 6 (manual)
User story: As a patient looking for something specific, I want to filter my recommendations by my location and availability, so that I can only see doctors tailored to my preferences.

Prerequisites (get the app running)
1. From the repository root, ensure the stack is up (see root README). Typical local URLs: web app http://localhost:5173, API http://localhost:3000.
2. Ensure the database is populated so searches return doctors (e.g. follow README “Database Setup For New Devs” using the checked-in data script if needed).
3. Use a desktop browser with JavaScript enabled. If the browser prompts for location access on the results page, you may allow it so distance next to each doctor’s address is meaningful; the test still works if you deny it.

Steps
1. Open the DocSeek home page in your browser (usually http://localhost:5173).
2. In the main symptom field, describe something specific you want care for (for example one of the suggested chips, or your own short description). Enter enough detail that validation allows you to continue (if the app asks for more detail, add a sentence and submit again until you proceed).
3. Scroll to the section titled “Filter by your preferences.”
4. In “Location (city, state, or ZIP),” enter a real area that should match doctors in your dataset (the placeholder suggests a format like “Pittsburgh, PA”). Leave other fields as needed for the next step.
5. Check the box “Only show doctors accepting new patients” if you want the list limited to doctors who are accepting new patients; leave it unchecked if you want no availability filter.
6. Submit the search (same control you use to run the doctor search from the home page) so you land on the “Recommended doctors” results page.
7. Confirm the results page shows your search symptoms and, when filters were applied, a line such as “Filtered by:” followed by your location and/or “Accepting new patients,” plus a “Refine filters” control.
8. On the first recommendation card, read the doctor’s practice location line. If you allowed browser location, note the distance shown next to the address; otherwise note that a distance may still appear using app fallback logic.
9. Note the availability line at the top of the card (“Accepting new patients” vs. “Check availability”) and confirm it matches your availability filter when that filter was turned on.
10. Click “See the next recommended doctor” (or the equivalent control) to move through at least one more recommendation and confirm each card still reflects the same filtered set.
11. Click “Refine filters.” Change the location string and/or toggle “Only show doctors accepting new patients,” then click “Apply filters.”
12. Confirm the URL or on-screen “Filtered by” summary updates, the list reloads, and the first recommendation again reflects your new constraints (or you see the empty state message if no doctors match).
13. Optional: Click “Start a new search,” return home, and repeat steps 2–6 with different location/availability choices to confirm behavior is consistent.

Expected result
You can constrain recommendations by location and by whether the doctor is accepting new patients before and after the initial search, see those constraints reflected on the results page, and browse recommendations that honor those preferences (or see a clear message when no doctor matches).
